# terrace-k-dashboard SPEC

## 概要
Terrace.Kチームの稼働状況をブラウザで一覧できる管理画面を作成する。

## 目的
- 各エージェントの現在の稼働状況をひと目で把握できるようにする
- 運用者がCronジョブとGatewayの状態をまとめて確認できるようにする
- 最小構成で早く使い始められるダッシュボードを用意する

## 最優先・最小構成
1. 各エージェントの稼働状態表示
   - 対象エージェント：澪・ユイ・ナナセ・レイン
   - 表示内容：オンライン / オフライン
2. 各エージェントのモデル表示
   - 各エージェントが現在利用しているモデル名を表示
3. Cronジョブ一覧
   - ジョブ名
   - 状態（有効/無効が取れるなら表示）
   - 最終実行時刻
4. Gateway状態確認
   - 稼働中 / 停止 / 異常 など、取得可能な範囲で状態表示
5. 可能なら追加する機能
   - UI上から指定Cronジョブを強制実行するボタン
   - 実行前に確認UIを入れる

## 想定画面構成
- ヘッダー
  - 画面タイトル
  - 最終更新時刻
  - 手動更新ボタン
- エージェント一覧セクション
  - 4人分のカードまたはテーブル
  - 各カードに名前、状態、モデルを表示
- Cronジョブ一覧セクション
  - 一覧テーブル
  - ジョブ名、最終実行時刻、必要なら手動実行ボタン
- Gateway状態セクション
  - ステータス表示カード
  - 必要に応じて簡単な補足情報

## データ方針
- まずはOpenClawまたは周辺で取得できる既存情報を優先利用する
- リアルタイム更新が難しい場合はポーリングで十分
- 初期版では永続化や履歴グラフは不要
- APIコスト可視化は不要

## デザイン方針
- ナナセはFigmaを使わず、テキストベースのDESIGNSPEC.mdを作成する
- 管理画面として見やすさ・情報優先・軽量さを重視する
- きよぴさんの好みに合わせ、具体的で把握しやすいUIを優先する

## 実装方針（補完仕様）
- 最小構成ではPCブラウザでの閲覧を優先する
- 1画面完結のダッシュボードとして構成する
- スマホ最適化は初期版では必須にしないが、極端に崩れない配慮は行う
- Cron強制実行機能は安全性確認を入れた上で、実現可能なら実装する

## 未確定・確認ポイント
- 各状態情報をどのAPI/コマンド/ログから取得するか
- Cron強制実行をどの方式で行うか
- 認証の必要性（初期版ではローカルまたは限定運用前提の可能性あり）
