## レイアウト構造

### グリッド

- デスクトップ（640px以上）：2列 × 2行 グリッド
- モバイル（640px以下）：1列に切り替え（CSS Grid `grid-template-columns: 1fr`）
- カード間ギャップ：20px
- 最大幅：960px（中央揃え）

### カード構成

```
┌──────────────────────────────┐
│ ▔▔▔▔▔▔▔▔ アクセントライン ▔▔▔▔  ← 3px top border（メンバーカラー）
│                              │
│  [アバター]  名前              │
│             役職ラベル         │
│                              │
│  ┌──────────────────────┐    │
│  │ タスク名         バッジ │    │ × 3件
│  └──────────────────────┘    │
└──────────────────────────────┘
```

#### アバター
- 丸形（`border-radius: 50%`）
- サイズ：56×56px
- アクセントカラーのボーダーリング（2px）
- 画像読み込み失敗時はイニシャルフォールバック表示（onerror対応）

#### 役職ラベル
- カプセル型バッジ（pill shape）
- アクセントカラーで文字・ボーダー・背景を統一

#### タスクアイテム
- サーフェス色（`#181c27`）の背景でカードと視覚的分離
- `padding: 9px 12px`、`border-radius: 8px`
- タスク名（左）＋ステータスバッジ（右）の横並びレイアウト

#### ホバーエフェクト
- カードが3px上方向にリフト（`translateY(-3px)`）
- メンバーカラーのグロウシャドウを表示

## デザイン上の判断メモ

- **ダークモード採用**：ダッシュボードは長時間参照するツールであることを想定し、目への負担を軽減するダークモードを選択。また、キャラクターのカラフルなアクセントカラーが映えやすいという意図もある。
- **メンバーカラーの統一的な使用**：アクセントカラーをカード全体で一貫して使うことで、「誰のカード」かを色だけでも即時に識別できるようにした。
- **フォールバックアバター**：onerrorハンドリングでイニシャル表示にフォールバック。ブラウザによっては絶対ファイルパスの読み込みが制限される場合があるため、サーバー経由での確認を推奨（ユイへの実装注意事項として共有）。
- **JSなし**：SPEC要件の「JavaScript不使用でも成立するシンプルな構成が望ましい」に従い、インタラクションをCSS onlyで実装。
